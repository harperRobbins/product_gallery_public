# 文档维护规范

更新时间：2026-04-07

## 1. 目标

确保系统行为、配置、部署、排障与文档始终一致，避免“代码已变更但文档滞后”。

## 2. 适用范围

以下变更都必须同步更新文档：
- 新增功能或页面
- 接口新增/修改/下线
- 字段、状态码、排序规则、筛选规则变化
- 数据库表结构变更
- 部署方式、端口、域名、服务名变化
- 配置项、环境变量、第三方依赖变化
- 已知问题处理策略变化

## 3. 必须更新的文件

最少更新：
- `docs/系统完整使用手册.md`
- `docs/变更记录.md`

按需更新：
- `README.md`（入口、快速启动、关键能力）
- 其它专题文档（若后续新增）

## 4. 更新流程（强制）

1. 开发前：确认是否影响文档
2. 开发中：记录“行为变化点”与“配置变化点”
3. 提交前：完成文档更新
4. 验收时：按“文档校验清单”逐条核对

## 5. 文档校验清单

每次提交请自检：
- 是否说明了功能入口（前台/后台路径）
- 是否说明了参数/字段变化
- 是否说明了状态与异常处理
- 是否说明了部署或配置影响
- 是否在 `docs/变更记录.md` 添加记录（日期、模块、变化、影响）
- 文档示例命令是否可执行

## 6. 变更记录格式（统一）

在 `docs/变更记录.md` 按以下模板新增：

```text
## YYYY-MM-DD
- 模块：
- 类型：新增 / 修改 / 修复 / 重构 / 运维
- 变更：
- 影响范围：
- 是否需要迁移：是/否（若是，写清步骤）
```

## 7. 责任约定

- 功能开发人：负责同步更新文档
- 代码审核人：负责检查文档是否同步
- 发布执行人：负责确认文档与线上状态一致

## 8. 发布门禁（建议执行）

未完成文档更新的变更不应发布到生产环境。
